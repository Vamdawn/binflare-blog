# Binflare Blog 文章列表与详情页重设计（Design）

- 日期：2026-03-13
- 状态：已评审通过（用户确认）
- 适用范围：`apps/web` 文章列表页（`/`）与文章详情页（`/posts/:slug`）

## 1. 背景与目标

当前文章列表与详情页样式偏基础模板风，信息层级与品牌辨识度不足。  
本次目标是在不改动路由与内容数据链路的前提下，抛弃现有视觉样式，建立统一且可扩展的阅读体验。

## 2. 设计决策（已确认）

### 2.1 总体方向

- 风格：新瑞士编辑风（Swiss Modernism 2.0）
- 列表密度：中密度（标题 + 日期 + 2~3 行摘要）
- 详情布局：纯阅读单栏
- 动效强度：轻动效（150~220ms）
- 最终方案：A（网格卡片列表 + 单栏阅读详情）

### 2.2 信息架构与页面骨架

#### 列表页 `/`

- 顶部：站点头部（站名 + 当前页入口）
- 主标题区：`文章` + 一句副标题
- 内容区：卡片网格（桌面双列，移动端单列）
- 卡片字段：标题、日期、摘要、`阅读全文`

#### 详情页 `/posts/:slug`

- 顶部信息行：返回列表 + 日期
- 标题区：主标题 + 可选 summary
- 正文区：单栏阅读容器（限定宽度，强调可读性）
- 底部：返回列表（不增加上一篇/下一篇）

### 2.3 视觉系统（Design Tokens）

#### 颜色

- `--bg: #F8FAFC`
- `--surface: #FFFFFF`
- `--text: #0F172A`
- `--text-muted: #64748B`
- `--border: #E2E8F0`
- `--accent: #2563EB`
- `--accent-strong: #1D4ED8`
- `--focus-ring: #93C5FD`

#### 字体

- 标题：`Newsreader`
- 正文与 UI：`Roboto`

#### 字号与节奏

- 正文：`18px`，`line-height: 1.75`
- 元信息：`14px`，`font-weight: 500`
- 间距体系：`4/8/12/16/24/32/48`
- 列表容器最大宽度：`1120px`
- 详情正文最大宽度：`720px`

### 2.4 交互与动效

- 卡片 hover：`translateY(-2px)` + 边框增强
- 链接 hover：下划线过渡
- 页面进入：轻量 `fade + up`
- 动画时长：`150~220ms`
- 支持 `prefers-reduced-motion`：降低或关闭位移动画

### 2.5 可访问性与质量底线

- 正文对比度满足 WCAG AA（>= 4.5:1）
- 键盘焦点可见（2px ring）
- 不仅用颜色表达状态（同时有形态变化）
- 小屏不出现横向滚动

## 3. 技术边界与不变项

- 保持现有路由结构：`/`、`/posts/:slug`、`/404`
- 保持文章数据来源：`getPosts()` 与 `getPostBySlug()`
- 保持 Markdown 渲染逻辑：仅升级样式，不变更渲染器行为
- 不在本期新增：搜索、标签、目录、推荐流、上一篇/下一篇

## 4. 组件影响面

- `SiteLayout`：重做头部与容器视觉
- `PostListPage`：增加页面级标题区与网格容器
- `PostCard`：重构信息层级与交互态
- `PostDetailPage`：重构文章头部信息区与正文容器
- `MarkdownRenderer`：增强排版样式（标题、段落、代码块、列表、引用）
- 样式文件：重建 token 与页面样式组织

## 5. 数据流与边界行为

- 列表页摘要优先级：`frontmatter.summary` > 正文截断（120 字）
- 无效 slug 或缺失 slug：继续跳转 `/404`
- 超长标题：列表卡片最多两行并省略
- 超长代码块：详情页允许横向滚动，不挤压主文本列

## 6. 测试策略（TDD 对齐）

先补/改测试，再进行样式与结构重构：

- 列表页：
  - 渲染文章卡片数量与标题链接
  - 摘要回退逻辑有效（无 summary 时使用正文摘取）
- 详情页：
  - 渲染标题、日期、返回链接、Markdown 主体
  - 缺失/错误 slug 按预期进入 404
- 可访问性基础断言：
  - heading 层级与主要 link 可被查询

## 7. 实施拆分原则

实现阶段按小步迭代进行，避免一次修改过多文件导致回归风险。  
优先顺序：样式 token 基建 -> 列表页 -> 详情页 -> Markdown 排版 -> 测试修正与验证。
